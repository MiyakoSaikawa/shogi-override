# 乱棋

乱棋は、通常の将棋に独自のルールを加えた、特殊ルール将棋アプリです。
通常の将棋では自分の駒のみを動かしますが、本アプリでは「相手の王将以外の駒も動かせる」というルールを採用しています。これにより、既存の将棋とは異なる読み合いや駆け引きが生まれ、従来の将棋を知っている人でも新鮮な感覚で遊べるゲームを目指しました。

## 制作背景

本アプリを制作したきっかけは、以前から自分でゲームを作ってみたいという思いがあったことです。私はWebアプリケーション開発に関心があり、単に画面を表示するだけでなく、ユーザーの操作に応じて状態が変化し、対戦相手との駆け引きが生まれるようなアプリを作りたいと考えていました。
また、私自身や友人は、既存の遊びのルールを少し変えたり、あえて本来のルールから逸脱した遊び方を考えたりすることが好きでした。既存のゲームに少し違うルールを加えるだけで、まったく別の戦略性や面白さが生まれる点に魅力を感じていました。
さらに、アルバイト先の学童で、子どもたちが今あるおもちゃや遊びに少しずつ飽き始めている様子を見たことも制作のきっかけの一つです。新しいおもちゃを用意するだけでなく、既存の遊びのルールを変えることで、新しい楽しみ方を生み出せるのではないかと考えました。
そこで、広く知られている将棋をベースにしながら、「相手の駒も動かせる」という独自ルールを加えたゲームを考案しました。既存の将棋の知識を活かしつつ、通常とは異なる判断が求められるため、友人同士や子どもたちでも新鮮な感覚で遊べるアプリを目指しました。

## 技術選定およびその意図

### Django

バックエンドにはDjangoを採用しました。
Djangoは、URLルーティング、テンプレート管理、ユーザー認証、セッション管理、管理画面など、Webアプリケーションに必要な機能が標準で充実しているフレームワークです。本アプリでは、ログイン・新規登録、オフライン対戦、オンライン部屋作成、ランダムマッチング、対戦結果の保存など、複数の機能を実装する必要があったため、これらを一貫して扱えるDjangoを選択しました。
特に、オンライン対戦ではルームIDやプレイヤー情報を管理する必要があります。これらをURLに直接含めると、URLを書き換えることで不正に参加できる可能性があるため、Djangoのセッションを利用してサーバー側で管理する設計にしました。これにより、ユーザーごとの対局状態を安全に扱えるようにしています。
また、Django Adminを利用することで、対戦結果を管理画面から確認できるようにしました。勝敗が確定した際に、対戦種別、プレイヤー名、勝者、敗者、勝敗理由、記録日時などを保存し、運用時に対戦履歴を確認できる構成にしています。

### Docker Compose

実行環境の管理にはDocker Composeを使用しました。
本アプリでは、Djangoアプリケーション、PostgreSQL、Nginxなど複数の要素を組み合わせて動作させる必要があります。Docker Composeを用いることで、それぞれの環境をコンテナとして分離し、設定ファイルからまとめて起動・管理できるようにしました。
これにより、ローカル環境とサーバー環境の差異を小さくし、環境構築やデプロイ時の再現性を高めることを意識しています。

### Nginx / Gunicorn

本番環境での公開を想定し、DjangoアプリケーションはGunicornで起動し、Nginxをリバースプロキシとして利用する構成にしています。
Djangoの開発用サーバーは本番運用には適していないため、アプリケーションサーバーとしてGunicornを使用しました。また、Nginxを前段に配置することで、外部からのリクエストをDjangoアプリケーションへ転送し、静的ファイル配信やHTTPS対応を行いやすい構成にしています。
