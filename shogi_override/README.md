# EX将棋 Django版

## 起動方法（pipenv）

```bash
cd ex_shogi_django
pipenv install -r requirements.txt
pipenv run python3 manage.py runserver
```

ブラウザで以下を開きます。

```text
http://127.0.0.1:8000/
```

## 対戦モード

### オフライン対戦

タイトル画面の「オフライン対戦」を押すと、1台の画面で黒・白が交互に操作できます。

### オンライン対戦

タイトル画面の「オンライン部屋作成」を押すと、黒番として部屋が作成されます。
画面上部に表示されるルームIDを相手に伝えてください。
相手はタイトル画面の「オンライン部屋に参加」欄へルームIDを入力すると、白番として参加できます。

現時点のオンライン同期はDjango標準のみで動くポーリング方式です。開発用としては動作しますが、実運用ではDB保存やWebSocket化を推奨します。


## オフライン対戦の状態管理

オフライン対戦は Django のセッションに保存されます。
そのため、同じサーバーに複数のブラウザでアクセスしても、それぞれのブラウザごとに別々の盤面状態が保持されます。

オンライン対戦は従来どおりルームIDごとの共有盤面を使います。


## 投了機能

対戦画面の「投了」ボタンから投了できます。オフライン対戦では現在の手番側が投了し、オンライン対戦ではボタンを押したプレイヤーが投了します。投了後は勝利ポップアップが表示され、リセットボタンで盤面を初期化できます。


## 勝敗後のタイトルボタン

勝利または投了後のポップアップには「リセット」と「タイトル」ボタンがあります。
「タイトル」を押すと、現在の盤面をリセットしてからタイトル画面 `/` に戻ります。


## 投了確認と投了リザルト

オンライン対戦中に「投了」またはナビゲーションの「タイトル」を押した場合、
「本当に投了しますか？」という確認が表示されます。

片方のプレイヤーが投了すると、相手側の画面にもポーリングによりリザルトが表示されます。
投了による勝利の場合、勝者側には「相手が投了しました」と表示されます。

## 先手・後手選択

タイトル画面で「オフライン対戦」または「オンライン部屋作成」を押すと、先手・後手・ランダムを選択できます。

- オフライン対戦: 選択した側が最初に指します。
- オンライン対戦: 部屋作成者の担当色を選択します。参加者は自動的に反対側になります。
- ランダム: サーバー側で黒・白のどちらかをランダムに決定します。

## 追加機能：ログイン必須のランダムオンライン対戦

タイトル画面に「オンライン対戦」を追加しました。

- オフライン対戦：ログイン不要
- オンライン部屋作成：ログイン不要
- オンライン部屋に参加：ログイン不要
- オンライン対戦：ログイン必須

「オンライン対戦」を押すと、ログイン済みユーザー同士でランダムにマッチングします。
未ログインの場合は、ログイン・新規登録ポップアップが表示されます。

## ログイン・新規登録

ナビゲーションの「ログイン」からログイン・新規登録できます。

新規登録項目：

- ユーザーネーム
- パスワード

既に存在するユーザーネームでは登録できず、「すでにその名前は使用されています。」と表示されます。

## 起動前の注意

Django標準のユーザー認証を使うため、初回起動前にマイグレーションを実行してください。

```bash
pipenv install -r requirements.txt
pipenv run python3 manage.py migrate
pipenv run python3 manage.py runserver
```

以降は以下で起動できます。

```bash
pipenv run python3 manage.py runserver
```

## ログイン・新規登録ページ方式

ログインと新規登録はポップアップではなく、専用ページで行います。

- `/login` : ログインページ
- `/register` : ユーザー新規登録ページ

ログインに成功するとタイトルページ `/` に戻ります。
ログインに失敗した場合は「ユーザー名またはパスワードが間違っています。」と表示されます。

新規登録に成功すると、`/login?registered=1` に移動し、ログインページ上に「登録成功」と表示されます。
既存ユーザー名と重複した場合は「すでにその名前は使用されています。」と表示され、登録されません。

オンライン対戦はログイン必須です。未ログイン状態でオンライン対戦を選ぶとログインページへ移動します。
オフライン対戦、オンライン部屋作成、オンライン部屋参加はログイン不要です。

## ランダムオンライン対戦の待機画面

「オンライン対戦」を押すと、盤面へ直接移動せず、待機画面へ遷移します。
待機中は「マッチング中です」と表示され、相手が見つかると「マッチングしました。」に変わり、2秒後に対戦画面へ移動します。

待機中に「キャンセル」を押すと、マッチングを取りやめてタイトル画面へ戻ります。


## 再戦機能

オンライン部屋作成・ランダムオンライン対戦では、勝敗決定後の「リセット」ボタンが「再戦する」になります。
片方が再戦を選ぶと「相手の選択を待っています」と表示され、もう片方には「相手が再戦を希望しています」と表示されます。
両者が再戦を選ぶと同じルーム・同じ担当色で盤面が初期化されます。
どちらかが先にタイトルを選んだ場合、その後に再戦を選んだ側には「再戦できませんでした、タイトルに戻ります。」と表示されます。

## ルーム管理の分離

この版では、オンライン部屋作成とランダムマッチングのオンライン対戦で、サーバー側のルーム管理を分離しています。

- オンライン部屋作成: `manual_rooms` で管理
- オンライン対戦（ランダムマッチング）: `match_rooms` で管理

そのため、同じ文字列のルームIDが発生しても、手動作成ルームとランダムマッチングルームが混ざらないようになっています。

URLも以下のように分かれています。

- オンライン部屋作成: `/online/<room_id>/<player>`
- オンライン対戦: `/match/<room_id>/<player>`

フロント側では `window.SHOGI_CONFIG.mode` により、以下のように区別しています。

- `room`: オンライン部屋作成
- `match`: ランダムマッチングのオンライン対戦
- `offline`: オフライン対戦

## セキュリティ改善: URLからルームID/プレイヤー情報を非表示化

オンライン部屋作成・オンライン対戦では、対局画面URLに `room_id` や `player` を含めないように変更しています。

### 変更前

```text
/online/<room_id>/<player>
/match/<room_id>/<player>
```

### 変更後

```text
/online/play
/match/play
/online/waiting
```

実際の `room_id` と `player` は Django の session に保存されます。
そのため、URLを書き換えて別プレイヤーになりすますことはできません。

また、`/move`, `/state`, `/reset`, `/surrender`, `/postgame-choice` などのAPIでも、フロントから送られた `room` や `player` を信用せず、サーバー側の session から参加ルームと担当色を判定します。


## 追加修正: オンライン対戦時のプレイヤー視点表示

オンライン部屋作成・オンライン対戦では、各ユーザーの画面で自分の駒が下側、相手の駒が上側になるように盤面を表示します。
また、持ち駒表示は左が相手、右が自分になります。ユーザー名が取得できる場合はユーザー名を表示し、取得できない場合は「あなた」「相手」と表示します。

オフライン対戦では従来どおり、左に白の持ち駒、右に黒の持ち駒を表示します。


## Django管理画面（DB確認）

管理画面を使う場合は、初回にマイグレーションと管理者ユーザー作成を行ってください。

```bash
pipenv run python3 manage.py migrate
pipenv run python3 manage.py createsuperuser
pipenv run python3 manage.py runserver
```

管理画面URL:

```text
http://127.0.0.1:8000/admin/
```

ユーザー情報はDjango標準の `Users` から確認できます。
セッション情報は `Sessions` から確認できます。ただし、このプロジェクトはブラウザごとの状態保持に signed cookie session を使用しているため、オフライン盤面などの一部セッション情報はDBの `Sessions` には保存されません。


## ログインが必要な機能

以下の機能はログインが必要です。

- オンライン対戦（ランダムマッチング）
- オンライン部屋作成

以下の機能はログイン不要で利用できます。

- オフライン対戦
- オンライン部屋に参加

## 一手ごとの持ち時間

- オフライン対戦とオンライン部屋作成では、開始時に一手ごとの持ち時間を選択できます。
- 選択肢は「無制限」または 10 秒〜120 秒までの 10 秒刻みです。
- ランダムマッチングのオンライン対戦は 90 秒固定です。
- 制限時間内に着手しない場合、その手番は時間切れとなり、相手の番に移ります。
- 同じ対局中に時間切れが累計 3 回になると敗北になります。

## 駒画像の配置

駒は `static/images/` に置いたPNG画像で表示されます。画像が存在しない場合は、従来どおり文字表示に自動フォールバックします。

対応ファイル名は以下です。

| 駒 | ファイル名 |
|---|---|
| 歩 | `hu.png` |
| 香 | `kyousha.png` |
| 桂 | `keima.png` |
| 銀 | `gin.png` |
| 金 | `kin.png` |
| 角 | `kaku.png` |
| 飛 | `hisha.png` |
| 先手の王 | `ou.png` |
| 後手の王 | `gyoku.png` |
| と | `tokin.png` |
| 成香 | `narikyou.png` |
| 成桂 | `narikei.png` |
| 成銀 | `narigin.png` |
| 馬 | `ryouma.png` |
| 龍 | `ryuuou.png` |

`gyoku.png` が存在しない場合、後手の王は `ou.png` にフォールバックします。
オンライン対戦では、自分の駒はそのまま、相手の駒は180度回転して表示されます。オフライン対戦では白側の駒が180度回転して表示されます。


## オンライン部屋作成の待機画面

オンライン部屋作成では、作成後すぐ盤面には移動せず、待機画面で相手の参加を待ちます。
待機画面には部屋IDが表示され、「コピー」ボタンでIDをクリップボードにコピーできます。
相手はタイトル画面の「オンライン部屋に参加」欄へその部屋IDを入力して参加します。
